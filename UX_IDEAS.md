# UX 개선 아이디어 (벤치마킹 수집)

> uxresearch 루프가 다른 게임/모바일 UX 패턴을 조사해 카우보이(cowboy_party)에 적용할 개선안을 모은다.
> **제안 전용** — 채택/기각·구현은 사용자/다른 세션 몫. 외부 상표·IP 베끼기 금지(일반 UX 패턴만).
> 형식: 사이클별 날짜·주제, 개선안마다 [무엇을 / 왜 / 어디 화면 / 우선순위 / 난이도 / 근거].

## 2026-06-17 · 주제: 첫 실행 온보딩 (First-Run Onboarding)

**현황**: 첫 실행 시 별도 온보딩 없이 바로 홈 메뉴(컴퓨터와 대결 / 온라인 대전 / 게임 방법) 노출.
규칙은 `how_to_play_screen.dart`의 긴 텍스트 카드 리스트(2~6명·턴 행동·장전·빵야·방어·슈퍼빵야·캐릭터 16종…)로만 전달. 신규 유저가 "읽어야" 룰을 안다.

**근거(조사)**: 모바일 게임 온보딩 정설 — "텍스트 벽 대신 직접 해보며 배우게"(learn-by-doing)가 리텐션을 최대 50%까지 끌어올린다. 핵심 메커닉만 먼저, 부가 요소는 점진 공개(gradual complexity). Among Us/Goose Goose Duck도 모드·역할이 많아(GGD 45역할/6모드) 첫 진입 혼란이 크고, 그래서 GGD는 "튜토리얼을 한두 번은 꼭 해보라"를 1순위 팁으로 둠. [출처: Adrian Crook 온보딩 가이드, Medium 모바일 온보딩 UX, GGD 비기너 가이드]

### 개선안
1. **[무엇을]** 첫 실행 1회 한정 "1턴 인터랙티브 튜토리얼" — 봇 1명과 1:1, 화면 위 화살표 코치마크로 ①장전 탭 → ②(다음 턴)빵야로 봇 조준 → ③봇이 쓰러지고 "승리!" 까지 30초 내 손으로 한 사이클 체험.
   **[왜]** "장전한 총알은 다음 턴부터 사용"이라는 카우보이의 핵심·헷갈리는 규칙을 글 대신 몸으로 학습. **[어디]** 첫 실행 시 home_screen 진입 직후(스킵 버튼 필수). **[우선순위]** 높음 **[난이도]** 중(기존 offline 엔진 재사용 + 코치마크 오버레이).

2. **[무엇을]** 홈 진입 시 "처음이신가요? 30초 튜토리얼 ▶" 펄스 배지를 '컴퓨터와 대결' 버튼 위에 1회 노출(완료/스킵하면 사라짐).
   **[왜]** 강제 풀스크린 튜토리얼은 이탈 유발 — 권유형이 덜 거슬리면서 발견성 확보. **[어디]** home_screen 상단 또는 첫 버튼. **[우선순위]** 중 **[난이도]** 하.

3. **[무엇을]** how_to_play를 "핵심 3장(원·턴 행동·장전→빵야) → 더보기(캐릭터/슈퍼빵야/덫·연막)" 2단 구조로 접기.
   **[왜]** 현재 16캐릭터+예외규칙까지 한 스크롤에 평면 나열 → 정보 위계 없음. 점진 공개로 첫 화면 인지부하 감소. **[어디]** how_to_play_screen. **[우선순위]** 중 **[난이도]** 하.

4. **[무엇을]** 슈퍼빵야·반응속도 결투처럼 "처음 발생하는 순간"에만 뜨는 1줄 툴팁(JIT, just-in-time 온보딩).
   **[왜]** 모든 규칙을 시작에 몰아주지 말고 마주치는 순간 설명 = gradual complexity의 정석. **[어디]** offline/online_game_screen 이벤트 트리거. **[우선순위]** 중 **[난이도]** 중.

5. **[무엇을]** 첫 게임 종료 후 "다음엔 온라인에서 친구와!" 1회 안내 + 방 코드 공유 동선 강조.
   **[왜]** 첫 세션 성공 직후가 소셜 전환 골든타임(리텐션). **[어디]** 결과 화면. **[우선순위]** 하 **[난이도]** 하.

## 2026-06-17 · 주제: 결과/승리 화면 & 공유 루프 (Match-End Screen)

**현황**: 결과는 `_resultCard` — 제목 한 줄("승리! 최후의 1인" 등) + [홈으로]/[다시하기] 버튼뿐. 온라인은 코인/미션 토스트만. **공유 동선·전적 요약·MVP/하이라이트·즉시 재대결(rematch)·친구 비교 전무.** 한 판의 드라마(누가 누굴 쐈나, 몇 턴, 슈퍼빵야 등)가 끝나면 사라짐.

**근거(조사)**: 카피캣 방지를 위해 일반 패턴만 — 매치 종료 화면은 (1)전적 리캡을 한 페이지로 응축, (2)MVP/베스트 순간을 애니메이션으로 치하(패자 팀에서도 MVP 가능), (3)재시도/다음판 빠른 액션, (4)친구 점수 대비·이긴 친구 표시로 공유 유발 → 바이럴. 공유는 "재미가 지속될 때"만 일어나므로 리텐션과 한 묶음. [출처: 모바일 바이럴 디자인(LinkedIn), 게임 특허의 친구-하이스코어 비교 패턴, THE FINALS/Marvel Rivals/Rematch 매치 리캡·MVP 화면 개편 사례]

### 개선안
6. **[무엇을]** 결과 카드에 **공유 버튼** 추가 — "카우보이에서 6인 중 최후 생존 🤠 #카우보이파티" 같은 한 줄 + (가능하면) 결과 카드 캡처를 `share_plus`로 시스템 공유.
   **[왜]** 현재 공유 동선 0 → 가장 싸고 효과 큰 바이럴 훅. **[어디]** offline/online `_resultCard`. **[우선순위]** 높음 **[난이도]** 하(텍스트 공유) / 중(이미지 캡처).

7. **[무엇을]** **원-탭 재대결(Rematch)** — 온라인 결과에서 "같은 방 인원으로 다시" 버튼(방장이 누르면 전원 재시작 동의 흐름). 오프라인은 이미 다시하기 있음 → 동일 인원·캐릭터로 즉시.
   **[왜]** "다음 판" 빠른 액션이 세션 길이·리텐션 핵심. 방 다시 만들기 마찰 제거. **[어디]** online_game_screen 결과. **[우선순위]** 높음 **[난이도]** 중(온라인 동기화).

8. **[무엇을]** **한 줄 매치 리캡** — "12턴 · 슈퍼빵야 1회 · 네가 처치 2명" 같은 통계 + "이번 판 MVP: ○○(최다 처치)" 1줄.
   **[왜]** 끝난 판에 의미·자랑거리 부여 → 공유 동기 + 다음 판 동기. 패자도 'MVP/하이라이트'로 기분 좋게. **[어디]** 결과 카드 본문(엔진이 이미 턴/처치 추적). **[우선순위]** 중 **[난이도]** 중.

9. **[무엇을]** 승리 시 **연출 강화** — 현재 텍스트+테두리색만. 카우보이 톤의 가벼운 컨페티/먼지/배지 1회 애니메이션 + 승리 효과음(이미 Sfx.win 존재) 싱크.
   **[왜]** "보상감 있는 종료"가 재대결률↑. 과하지 않게(스킵 가능). **[어디]** 결과 카드 등장 모션. **[우선순위]** 중 **[난이도]** 하~중.

10. **[무엇을]** 온라인 결과에 **시즌 랭킹 변화 미리보기** — "+40점 · 이번 시즌 14위 → 11위" 한 줄(로그인 시).
   **[왜]** 진행감(progression)을 종료 화면에 응축하는 최신 트렌드(Rematch/THE FINALS). 다음 판 복귀 유인. **[어디]** online 결과 카드. **[우선순위]** 중 **[난이도]** 중(랭킹 델타 계산).

## 2026-06-17 · 주제: 온라인 로비/방 입장 마찰 (Lobby & Join Friction)

**현황**: play_tab = 빠른시작(배너) + [컴퓨터와 대결]/[방 만들기] + 게임방법 배너 + 공개 방 목록(제목·방장·n/6·입장/관전/꽉참). 방 만들기는 `online_lobby_screen`(제목·공개/비공개·비번 → 항상 6인방 생성). **방 코드 직접 입력칸이 play_tab에 없음**(비공개 방은 초대 링크로만 입장 유도). 방 목록은 당겨서 새로고침/수동 새로고침(실시간 자동갱신 아님). 빈 목록엔 empty state 있음(양호).

**근거(조사)**: 로비의 본질 = "나에게 맞는 방을 빠르게 찾아 들어가게". 베스트프랙티스 — (1)퀵조인 + (2)공유된 ID/코드로 직접 입장 둘 다 제공, (3)방 상태(인원·곧 시작 등)를 한눈에, (4)로비가 **실시간으로** 방 생성/삭제/인원 변동 반영, (5)방 만든 사람이 혼자 덩그러니 느끼지 않게(isolation 방지). [출처: SmartFoxServer 로비 문서, RIVEN 로비 설계 가이드, 멀티플레이 로비 devlog들]

### 개선안
11. **[무엇을]** play_tab에 **"코드로 입장" 입력칸**(6자리) 상시 노출 — 빠른시작 바로 아래 한 줄. 입력 즉시 `joinRoom`.
   **[왜]** 친구가 코드를 불러줘도 현재 직접 입력 동선이 약함(초대 링크 의존). 코드 입장은 로비 2대 진입로 중 하나. **[어디]** play_tab. **[우선순위]** 높음 **[난이도]** 하(joinRoom 이미 존재).

12. **[무엇을]** 공개 방 목록 **실시간 자동 갱신** — 현재 수동/당겨서만. 가벼운 폴링(예: 5초) 또는 RTDB 구독으로 인원·신규 방 자동 반영(화면에 있을 때만).
   **[왜]** "지금 들어갈 수 있는 방"이 실시간이 아니면 입장 실패·이탈. 로비 실시간 갱신은 기본기. **[어디]** play_tab `_rooms`. **[우선순위]** 중 **[난이도]** 중(배터리/읽기량 고려해 화면 보일 때만).

13. **[무엇을]** **혼자 만든 방 isolation 방지** — 방 생성 직후 대기실에서 "친구 초대(링크/코드 공유)" 버튼을 가장 크게 + "봇 채우고 시작" 옵션 제시.
   **[왜]** 공개해도 아무도 안 오면 방장이 혼자 기다리다 이탈 = 빈 로비 악순환. 즉시 봇으로 시작 가능하면 마찰 0. **[어디]** online_game_screen 대기실. **[우선순위]** 높음 **[난이도]** 중.

14. **[무엇을]** 공개 방 카드에 **맥락 정보** 보강 — 방장 캐릭터 아이콘, "곧 시작"(정원 임박) 배지, 평균/대표 랭킹대 표시.
   **[왜]** "어느 방에 들어갈지"를 한눈에. 곧 시작 방은 바로 합류 유인. **[어디]** play_tab `_roomRow`. **[우선순위]** 중 **[난이도]** 하~중(데이터 노출 범위 확인 필요).

15. **[무엇을]** 빠른 시작 매칭 화면에 **대기 중 미니 활동** — 단순 카운트다운만 있는 현재 화면에 "들어온 사람: ○○ 합류!" 실시간 리스트 + 팁 회전.
   **[왜]** 20초 빈 대기의 체감시간↓, 매칭 진행감↑(곧 채워질 거란 신뢰). **[어디]** matchmaking_screen. **[우선순위]** 하 **[난이도]** 하.

## 2026-06-17 · 주제: 캐릭터 수집·해금 동기 (Collection & Unlock Motivation)

**현황**: 상점 탭 = 16캐릭터 평면 2열 그리드(각 카드: 초상·이름·능력·잠김/코인가격·장착/체험). 코인은 승리·출석으로 모음. ???(mystery)는 나머지 전부 보유 시 해금. **수집 진행도(컬렉션 미터)·등급(rarity) 구분·"다음 추천 해금"·완성 보상·해금 순간 연출이 없음.** 능력 텍스트가 카드마다 빽빽(정보 평면).

**근거(조사)**: 캐릭터 수집 메타의 리텐션 핵심 — (1)진행/성장(해금·강화)이 강한 동기, 컬렉션에 정서적 애착 형성, (2)희귀도(rarity)로 가치·열망 부여, (3)**명확한 진행 표시**(다음 목표까지 거리: 미터·별·%)는 인간이 꼭 필요로 함, (4)컬렉션 100% 완성 시 독점 보상(프레임·쿠폰 등), (5)완성도를 타인과 비교. [출처: Udonis 캐릭터 컬렉션 메타, GameRefinery 컬렉션 시스템, GDevelop 캐주얼 게임 루프]

### 개선안
16. **[무엇을]** 상점 상단에 **컬렉션 진행 미터** — "16명 중 7명 보유 (44%)" 진행바 + 보유/장착 요약.
   **[왜]** "다음 목표까지 거리"를 보여주는 진행 표시는 수집 동기의 기본. 지금은 전체 진척을 한눈에 못 봄. **[어디]** characters_tab 상단 sliver. **[우선순위]** 높음 **[난이도]** 하.

17. **[무엇을]** **컬렉션 완성 보상** — 전 캐릭터(또는 ??? 포함) 수집 시 독점 보상(전용 칭호/프로필 프레임/이펙트). ???가 사실상 완성 트로피지만 "보상"으로 명시·축하 연출.
   **[왜]** 장기 목표 완성에 보상이 있어야 끝까지 모음. 현재 ???는 잠금 해제일 뿐 완성 축하·보상감이 약함. **[어디]** characters_tab + Meta. **[우선순위]** 중 **[난이도]** 중.

18. **[무엇을]** **"다음 해금 추천"** 카드 — 현재 코인으로 살 수 있는 것 중 한 명을 "지금 해금 가능!"으로 강조, 부족하면 "앞으로 N코인"·근접 캐릭터 상단 정렬.
   **[왜]** 16개 평면 그리드에서 "뭘 다음에?" 결정 마찰 큼. 가까운 목표 제시 = 행동 유도. **[어디]** characters_tab. **[우선순위]** 중 **[난이도]** 하~중.

## 2026-06-17 · 주제: 접근성·사운드/햅틱 (Accessibility & Audio/Haptic Feedback)

**현황**: 색으로 상태/액션 구분(danger=빨강, sage=초록, gold 등), 효과음(Sfx) + 일부 햅틱(HapticFeedback) 사용. **색 외 보조 신호(아이콘/패턴)가 결과 전달에 충분한지 점검 필요, 텍스트 크기 조절·고대비 모드·사운드/햅틱 on-off 설정 노출 여부 불명, 좌석 탭 등 터치 타깃 최소 크기(44pt), 색맹 모드 부재 가능성.**

**근거(조사)**: 색맹은 색만으로 정보 전달 금지 — **패턴/심볼 병행**. 본문 16px+·고대비·텍스트 크기 조절. **햅틱은 청각 대체**(난청 사용자에 소리 정보 전달), 터치 타깃 최소 **44×44pt**+충분한 간격으로 오탭 방지. 사운드/햅틱/자막 등은 사용자가 켜고 끌 수 있게. 접근성은 장애 사용자뿐 아니라 전체 사용성↑. [출처: SDLC 모바일 게임 접근성, iOS 접근성 가이드 2025, Accessibly/Keywords Studios]

### 개선안
41. **[무엇을]** **색+심볼 이중 표기** — 명중/탈락=💀 또는 X 아이콘, 방어 성공=🛡, 장전=🔄 등 색에 아이콘/패턴 항상 동반(현재도 일부 아이콘 있음 → 결과 상태 전반으로 일관 적용).
   **[왜]** 색맹 사용자가 빨강/초록만으론 결과 구분 난해. 심볼 병행이 정설이자 모두에게 명확. **[어디]** TableSeat 리빌·배너·결과. **[우선순위]** 중 **[난이도]** 하.

42. **[무엇을]** **설정에 사운드·햅틱·효과음 토글** + 텍스트 크기/고대비 옵션. (설정 화면 위치 확인 후.)
   **[왜]** 사용자별 on-off는 접근성 기본. 진동 민감/무음 환경/저시력 모두 대응. **[어디]** 설정 화면. **[우선순위]** 중 **[난이도]** 중(설정·환경설정 저장).

43. **[무엇을]** **햅틱을 결과 신호로 확장** — 명중당함=강한 진동, 방어 성공=가벼운 탁, 내 탈락=긴 진동 등 패턴화(소리 못 들어도 결과 전달).
   **[왜]** 햅틱은 청각 대체 채널. 무음/난청에서도 게임 결과 인지. 이미 HapticFeedback 일부 사용 → 의미별 패턴 체계화. **[어디]** 결과 처리 공통. **[우선순위]** 하 **[난이도]** 하.

   **[왜]** 강한 플래시는 전정과민/광과민 사용자에 부담. 접근성+편의. **[어디]** 설정 + 연출 분기. **[우선순위]** 하 **[난이도]** 하~중.

## 2026-06-17 · 주제: 소셜 초대·공유 바이럴 루프 (Invite & Viral Loop)

**현황**: 이미 양호한 기반 — `share_plus` 네이티브 공유 시트(F4, 실패 시 링크 복사 폴백), 초대 링크 `webBaseUrl?room=CODE` 딥링크(main.dart 처리), 대기실 "친구 초대" 버튼. **단, 공유 진입점이 주로 방 안/대기실에 한정(결과·홈 등 자연스러운 순간 부족), 초대 보상(double-sided)·초대 진척/티어 없음, 공유 콘텐츠가 텍스트 링크 위주(승리 이미지 카드 없음), 딥링크로 들어온 신규의 첫 경험 최적화 점검 필요.**

**근거(조사)**: 바이럴은 **K-Factor**(받은 초대당 전환 × 보낸 초대 수). >1이면 지수성장. 가장 효과적 추천은 **양방향 보상**(초대자+신규 둘 다), **티어 보상**(많이 초대할수록 좋은 보상=게임화), **여정 초반에 초대 유도**, **한정 기간 긴급성**, **마찰 없는 공유 UX**. 초대 배치(사용자가 가장 자연스럽게 공유를 만나는 곳)가 중요. [출처: Viral Loops 추천 프로그램·K-factor, Kurve K-Factor, CleverTap 바이럴 루프]

### 개선안
46. **[무엇을]** **공유 진입점 확대** — 결과 화면(승리 직후·#6과 연계), 홈, 빈 로비, 첫 게임 종료(#5)에 "친구 불러 같이 하기" 일관 버튼. 방 안에만 두지 말기.
   **[왜]** 초대 배치=바이럴 핵심. 승리 직후 등 감정 고조 순간이 공유율 최고. 현재는 대기실 중심. **[어디]** 결과/홈/play_tab. **[우선순위]** 높음 **[난이도]** 하(이미 _shareInvite 존재).

47. **[무엇을]** **양방향 초대 보상** — 초대 링크로 신규가 첫 게임 완료 시 초대자+신규 둘 다 코인. (어뷰징 방지: 신규 계정·1회 한정.)
   **[왜]** double-sided 보상이 추천 효율 최고. 무보상 공유는 K-factor 낮음. **[어디]** 딥링크 가입/첫판 완료 처리 + Meta. **[우선순위]** 높음 **[난이도]** 상(신규 판별·어뷰징 방지 — 사용자 결정 필요).

48. **[무엇을]** **승리 공유 이미지 카드** — 텍스트 링크만이 아니라 "🤠 6인 중 최후 생존! 카우보이파티"가 박힌 결과 카드 이미지를 함께 공유(#6 공유버튼과 결합).
   **[왜]** 이미지가 텍스트보다 클릭·전환↑(비플레이어도 보고 흥미). 공유 콘텐츠 품질이 K-factor에 직결. **[어디]** 결과 카드 캡처 + Share. **[우선순위]** 중 **[난이도]** 중(위젯→이미지 렌더링).

49. **[무엇을]** **초대 진척/미션** — "친구 3명 초대 시 캐릭터 변경권" 같은 티어형 초대 보상 + 진행바.
   **[왜]** 티어 보상이 지속 공유 동기(게임화). 1회성 공유보다 반복 유발. **[어디]** rewards_tab 또는 전용 초대 화면. **[우선순위]** 중 **[난이도]** 중.

50. **[무엇을]** **딥링크 신규 첫 경험 최적화** — `?room=CODE`로 들어온 사람이 닉네임 미설정/앱 첫 실행이어도 매끄럽게 방에 합류(닉 빠른 설정→바로 입장), 온보딩(#1)과 충돌 없이.
