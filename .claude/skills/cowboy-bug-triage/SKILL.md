# cowboy-bug-triage — 카우보이 버그 분류

Steinberger의 `github-project-triage`를 이 프로젝트에 맞춘 버전. 목적: 흩어진 버그
리포트(사용자 QA 메모, ntfy `cowboy-feedback-doonghwi` 제보)를 **하나의 우선순위 큐**로 만든다.

## 입력 소스
1. `cowboy_party/BUGS.md` — 사용자가 직접 적는 버그 큐(아래 형식).
2. ntfy 제보(`cowboy-feedback-doonghwi`) — 사용자가 캡처해 BUGS.md에 붙여넣음.
3. 세션 중 발견한 회귀.

## BUGS.md 항목 형식
```
- [ ] (P1) 제목 — 재현: <단계> / 기대: <정상> / 실제: <버그> / 화면: <파일 힌트>
```
- 상태: `[ ]` 대기 · `[~]` 진행 중 · `[x]` 완료(배포까지).
- 우선순위: **P0** 크래시·진행불가 · **P1** 핵심 기능 오작동 · **P2** UI/표현 · **P3** 사소.

## 분류 절차
1. 새 리포트를 위 형식으로 정규화(재현·기대·실제·화면 힌트 채우기). 정보가 모자라면
   코드를 빠르게 훑어(grep) 화면/파일을 추정해 힌트에 적는다.
2. 우선순위 매기기: 크래시/온라인 동기화 깨짐 = P0, 규칙/판정 오류·결제/닉네임 = P1.
3. 중복 병합, 한 줄에 한 버그.
4. P0→P1→P2→P3, 같은 우선순위면 영향 범위 큰 것 먼저로 정렬.
5. 결과를 BUGS.md에 다시 쓴다(완료 항목은 맨 아래로).

## 산출물
정렬된 BUGS.md + "다음 처리 후보 3개" 요약. 실제 수정은 **cowboy-fix-loop**가 한다.
