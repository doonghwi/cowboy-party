## A. Google Play — 데이터 안전 (Data safety)

### 1) 데이터 수집·공유 여부
- 데이터를 **수집함(Collected)**: 예 (아래 항목).
- 데이터를 제3자와 **공유(Shared)**: 아니오 (Firebase는 처리 위탁 인프라이며 별도 판매·공유 아님).
- 전송 중 **암호화(Encryption in transit)**: 예 (HTTPS).
- 사용자가 **삭제 요청** 가능: 예 (이메일/앱 내 제보 채널).

### 2) 수집 항목 매핑 (Play 데이터 유형 → 본 앱)
| Play 데이터 유형 | 수집? | 수집 항목 | 목적 | 필수/선택 |
|---|---|---|---|---|
| 개인 정보 > 이름 | 예 | **닉네임(사용자 지정 표시명)** | 앱 기능(플레이어 구분) | 필수 |
| 앱 활동 > 인앱 검색/상호작용 | 예 | 게임 플레이 기록(승/패, 코인, 출석) | 앱 기능, 분석(밸런스 통계) | 필수 |
| 앱 정보 및 성능 | 선택 | 사용량 카운트(실행 횟수 등 익명 집계) | 분석 | 선택 |
| 기기/기타 ID | 예 | **익명/계정 식별자(uid)** | 앱 기능(기록 연동), 계정 관리 | 필수 |

### 3) 수집 안 함(명시적으로 "아니오")
- 정확/대략적 위치, 이메일 주소·전화번호, 연락처, 사진/동영상, 음성, 캘린더,
  금융 정보, 건강/피트니스, 웹 검색기록, **광고 ID(AAID)**, 기기 연락처.

### 4) 보안 관행
- 전송 중 암호화: 예. 사용자 데이터 삭제 요청 메커니즘: 예(아래 연락처).
- 광고 SDK: 없음. 분석 SDK: **Firebase Analytics**(v12+, 익명 사용 통계 — 광고 ID 미사용, 아래 5 참고).

### 5) 광고 ID 선언 (앱 콘텐츠 > 광고 ID) — v13 기준 답안
- **"앱이 광고 ID를 사용하나요?" → 아니오(사용 안 함)**
- 근거: v12에서 firebase_analytics가 매니페스트에 `com.google.android.gms.permission.AD_ID`를
  자동 병합해 선언 요구가 떴음 → v13에서 **권한을 `tools:node="remove"`로 제거**하고
  `google_analytics_adid_collection_enabled=false` 메타데이터로 수집도 껐다(AndroidManifest.xml).
  리텐션·이벤트 측정은 앱 인스턴스 ID로 동작하므로 기능 손실 없음.
- ⚠️ 이 선언은 **매니페스트와 일치해야** 한다: "사용 안 함"으로 답하려면 반드시 **v13 이상**(AD_ID 제거본)을
  업로드해야 하며, v12(AD_ID 포함)를 올린 채 "사용 안 함"을 고르면 콘솔이 오류를 띄운다.
- 데이터 안전 양식의 "기기 또는 기타 ID"는 기존 답(uid 수집) 유지 — 광고 ID 항목은 계속 "수집 안 함".

## B. Apple — App Privacy (App Store Connect)

### 1) "데이터를 수집하나요?" → 예

### 2) 수집 데이터 → **사용자에게 연결됨(Linked)** 처리 (uid 기준)
| Apple 카테고리 | 데이터 타입 | 목적(Purpose) | 추적(Tracking)? |
|---|---|---|---|
| Identifiers | User ID (익명/계정 uid) | App Functionality | 아니오 |
| User Content | Other (닉네임) | App Functionality | 아니오 |
| Usage Data | Product Interaction (플레이 기록·승률) | App Functionality, Analytics | 아니오 |

### 3) Tracking(앱·웹 교차 추적)
- **아니오** — 다른 회사의 앱/사이트와 데이터를 연결해 광고/추적하지 않음. (ATT 프롬프트 불필요)

### 4) 수집 안 함
- 연락처 정보(이메일/전화/주소), 위치, 금융, 건강, 검색기록, 민감정보, 광고 데이터.
